# Current Feature Inventory

Audit date: 2026-07-23  
Scope and method: repository implementation was inspected end to end. Documentation, labels, models, routes and tests were not treated as proof without reachable working behavior. Status meanings are exactly those supplied in the audit request. This inventory should be read with [01-architecture-and-current-state.md](./01-architecture-and-current-state.md).

## Status definitions

- **COMPLETE:** implemented end to end, accessible to users, persistent, error-handled and reasonably tested.
- **PARTIAL:** functional implementation exists, but important workflow, reliability, security, UI or testing work is missing.
- **STUB:** only a route, model, UI, interface, placeholder or mocked/disconnected behavior exists.
- **ABSENT:** no meaningful implementation found.
- **UNKNOWN:** repository evidence is insufficient.

## Executive inventory summary

The repository provides a functional, narrow Admin-only MVP for deploying pre-compatible Django/React Docker Compose repositories to pre-provisioned SSH servers. Its strongest complete areas are remote server registration, generic Git/deploy-key operations, manual asynchronous deployment state/history, Compose lifecycle controls, encrypted project variables, JWT administration, and REST APIs. Coolify-class environment/tenant abstractions, ingress/TLS, managed databases, storage lifecycle, templates, monitoring, notifications, scheduling, backups and platform upgrades are largely absent.
